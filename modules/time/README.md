# time

## 역할

공통 시간 모듈이다.
서비스 전반에서 동일한 기준 시간대와 `Clock` 빈을 사용하도록 맞춘다.

## 제공 기능

- `Clock` 빈 auto-configuration
- `time.timezone` 기반 typed properties
- 기본 타임존 설정

## 방향성

- 서비스에서 `LocalDateTime.now()` 를 직접 남발하지 않고 `Clock` 주입을 우선한다.
- 테스트 가능성과 시간 기준 일관성을 높이는 것이 목적이다.

## 넣지 않을 것

- 비즈니스 스케줄 정책
- 배치 주기 계산 로직
- 서비스별 시간 규칙

## 사용 대상

- 시간 의존 로직이 있는 모든 서비스
- 테스트에서 시간 제어가 필요한 서비스
